sofarpc-cli 原型图

基于当前仓库的 README、架构文档、MCP 工具入口和 MCPB 分发方案整理。原型聚焦 agent 使用 SOFARPC 泛化调用的核心路径，不假设已有 Web UI。

README.md / README.zh-CN.md docs/architecture.md cmd/sofarpc-mcp internal/mcp cmd/sofarpc-mcp/skill scripts/build-mcpb.sh

项目分析结论

1 产品形态是 agent-first 本地 MCP Server，核心不是人工点击 UI，而是让 Agent 调用一组 typed JSON tools。
2 真实用户流程分两层：用户级 setup 注册 Claude Code / Codex；项目级 setup 写入 `.sofarpc/config*.json`。
3 调用主路径是 open -> target -> describe -> invoke dry-run -> invoke/replay；失败时跟随 errcode hint 到 doctor。
4 安全边界很重要：真实调用默认关闭，服务、target host、args 文件根目录和 response size 都有 guardrail。
5 MCP 设计已经补上 metadata、structured JSON mirror、resources、progress/logging；分发路径也要覆盖 macOS / Linux / Windows MCPB。

目标：把单个二进制注册到 Agent 客户端，并安装 embedded skill。

fresh machine
$ go install github.com/hex1n/sofarpc-cli/cmd/sofarpc-mcp@latest
$ open dist/mcpb/sofarpc-mcp-0.1.0-windows-amd64.mcpb

{ "version": "v0.1.x", "commit": "..." }
